# Go Module Updater CLI

Утилита командной строки для анализа зависимостей Go-модулей.  
Клонирует Git-репозиторий, читает go.mod и показывает, какие зависимости можно обновить.

## Возможности

- Анализ любого публичного Git-репозитория с Go-проектом
- Вывод имени модуля и версии Go
- Список всех зависимостей с текущими и доступными версиями
- Отображение прямых и косвенных зависимостей
- Сохранение результатов в JSON

## Что нужно для работы

- Go версии 1.21 или новее
- Git (любая актуальная версия)
- Доступ в интернет

## Как установить

### 1. Скачайте проект
```bash
git clone https://github.com/FallenKnightW8/go-mod-updater.git
cd go-mod-updater
2. Соберите программу
bash
go build -o go-mod-updater.exe
3. Проверьте, что всё работает
bash
go-mod-updater.exe --help
Как настроить список репозиториев
Создайте файл repos.txt
В той же папке, где лежит программа, создайте текстовый файл repos.txt.
В нём укажите репозитории, которые хотите проверять — по одному на строке.

Примеры форматов (все рабочие):

text
gin-gonic/gin
gorilla/mux
https://github.com/labstack/echo
git@github.com:gofiber/fiber.git
Пример заполненного файла:

text
gin-gonic/gin
gorilla/mux
prometheus/prometheus
kubernetes/kubernetes
Важные моменты:

Для GitHub достаточно указать user/repo

Можно использовать полные URL для любых Git-серверов

Каждый репозиторий с новой строки

Пустые строки не учитываются

Как пользоваться
Запуск
bash
analyze.bat
Выбор репозитория
Откроется меню со списком репозиториев из вашего файла

Введите номер нужного репозитория

Нажмите Enter

Вывод результата
После выбора репозитория у вас два варианта:

Нажать Enter — результат отобразится в виде таблицы прямо в консоли

Ввести 1 — результат сохранится в JSON-файл в папке results

Возврат в меню
После завершения анализа нажмите любую клавишу, чтобы вернуться к выбору репозитория.

Выход
В главном меню введите 0.

Что означает вывод
В таблице показывается:
Столбец	Что означает
Dependency	Название пакета
Current	Установленная версия
Latest	Доступная версия (если есть обновление)
Status	Up to date — всё актуально, Update available — можно обновить
Type	direct — прямая зависимость, indirect — косвенная
JSON-файл содержит:
json
{
  "module_name": "github.com/gin-gonic/gin",
  "go_version": "1.21",
  "dependencies": [
    {
      "name": "github.com/gin-contrib/sse",
      "current_version": "v0.1.0",
      "latest_version": "v0.2.0",
      "is_direct": true,
      "can_update": true
    }
  ]
}
JSON-файлы сохраняются в папку results с понятными именами:
results/gin-gonic_gin_13-07-26_14-30.json

Если что-то пошло не так
repos.txt not found
Программа не нашла файл со списком репозиториев.
Она автоматически создаст пример — отредактируйте его под себя и запустите заново.

git clone failed
Не удалось склонировать репозиторий. Возможные причины:

Не установлен Git — скачайте с git-scm.com

Нет интернета или проблема с доступом к серверу

Неверно указан URL репозитория

go.mod not found
Репозиторий клонирован, но это не Go-проект.
Утилита работает только с проектами, где в корне есть файл go.mod.

go list failed
Проблема с окружением Go.
Проверьте установку командой go version. Требуется версия 1.21 или новее.

Использование без меню
Можно работать с программой напрямую, без bat-файла:

bash
# Обычный анализ
go-mod-updater.exe --repo https://github.com/gin-gonic/gin

# Только прямые зависимости
go-mod-updater.exe --repo https://github.com/gin-gonic/gin --direct

# Вывод в JSON в консоль
go-mod-updater.exe --repo https://github.com/gin-gonic/gin --json
Как удалить
bash
del go-mod-updater.exe
del analyze.bat
del repos.txt
rmdir /s /q results
Что внутри

Клонирование: поверхностное (--depth 1), только последний коммит
Временные файлы: удаляются автоматически после анализа
Кодировка: UTF-8
